CHAPTER 01

架构

纯视觉智能体 · 三层进程模型 · 两条公开 LLM 通道

Lucid 不依赖 MCP、不需要插件、不调用应用 API。模型采用 Claude Opus 4.x，公开走两条通道，另有一条仅开发用的内部代理。

Anthropic API · claude-opus-4-5-20250929 GitHub Copilot OAuth · claude-opus-4-6 内部代理（仅开发）

三层进程模型

GUI · Tauri WebView

SvelteKit 页面：对话、计划、模板、记忆、打盹、设置。

Rust 外壳 · Tauri main

Sidecar 生命周期、系统托盘、全局热键、DPAPI 加密凭据。

引擎 · lucid.exe

Python 3.14 sidecar：ReAct 循环、截图金字塔、任务栏、调度、打盹、记忆。

UI ↔ 引擎走 stdio 上的 JSON-RPC。用户数据集中存在 ~/.lucid/（配置、日志、计划、记忆、图标、令牌）。

为什么坚持纯视觉

零接入成本

私有 ERP、RDP 会话、老 Win32、甚至游戏都能跑。

人在环中的安全

三档自主级：full / confirm_critical / confirm_each，叠加紧急热键与 PyAutoGUI fail-safe。

所见即所及

比专用 API 慢一些，但接入不会过期、不会断版。

ReAct 循环

observe / 观察
按模型当下选择的等级（L0–L3）截屏。
think / 思考
Claude 返回 tool_use：computer、launch_app、region……
safety / 安全
安全层过滤危险词、坐标、HITL 闸门。
act / 执行
经 PyAutoGUI / Win32 SendInput / 剪贴板下发。
verify / 核验
若开启 click_verify_enabled，事后取 L3 做像素 diff。
repeat / 重复
直到 finish() 或预算用尽。
CHAPTER 02

截图流水线

四个等级 · token 预算可控 · 像素级点击核验

图像很贵——不点名不送。系统教模型："看不到东西很正常；要看就主动调 screenshot()。"只有少数高价值动作会顺带挂图。

四个等级（L0–L3）

等级范围触发用途
L0图标库screenshot(level="icon_atlas")识别未知托盘 / 任务栏图标
L1整个虚拟桌面模型主动跨应用态势感知
L2活动窗口客户区launch_app / focus_window 顺带一次表单、菜单、列表操作
L3光标周围 ~210×210，绿色十字点击核验、模型主动像素级点击确认、tooltip / 小图标

关键取舍

不再事后自动挂图

2026-05 移除。长会话省 30–50% token。

L2 坐标网格

只在 L2 叠网格；系统提示教模型直接读像素坐标。

L3 定尺寸 + 十字

取消 UIA 自适应；尺寸稳定，模型的坐标心智才稳定。

点击核验

每次点击前后各取一张 L3；pixel_change_ratio < 0.5% 即提示"可能点空"。

重压缩预算

历史截图缩放 + JPEG 重压，再替换成文字占位。强制"同回合内转写关键信息"。

DPI 感知坐标

Per-Monitor V2，mss 物理像素，活动应用钉死，防焦点漂移。

CHAPTER 03

任务栏通知监听

两条并行通道 · 先到者胜

Action Center（WinRT UserNotificationListener）在 Teams / 微信上不可靠，因此 Lucid 启了两条并行通道。

通道 A — UIA（默认 · 事件驱动） 已落地

属性变化事件

订阅 Shell_TrayWnd 子树的 Name / HelpText 变化。

零轮询零 token

事件本身告诉我们"哪个应用"、"变了什么"。

冷却 + 快照清扫

每个应用单独冷却，避免同一通知反复触发。

lucid/taskbar_uia.py · 配置位于 [taskbar_uia]

通道 B — 视觉两步（兜底） 已落地

Diff
每 poll_interval_sec 抓底部居中区（约底 120 px，x ∈ 25%–75%）。dHash / 像素 diff；阈值以下直接收工。
LLM 确认
真有变化时，附上裁剪片，问"是不是新消息？哪个应用？"，输出结构化。

X 方向投影分段隔离变化列，避免时间/天气小部件误报。

仲裁

  • UIA 先确认时，第二步 LLM 被压制 visual_suppress_after_uia_sec（默认 20 秒）。
  • 确认事件经 strict 模式队列去重，可触发 auto_chat_instruction 自动回复任务。
  • 每个应用的通道偏好存在 taskbar_sources.json——打盹会学习并写回（见第 4 章）。
CHAPTER 04

打盹学习

空闲时低优先反思 · 学 tip、图标、任务栏小部件

v0 文本 — 已落地  v1 图标 — 已落地  v1 任务栏视觉 — 已落地  v2 心跳 — 计划中

什么时候会醒？

每 tick_interval_sec（默认 60 秒）worker 检查四个条件，必须全部满足：

没在跑 worker

打盹绝不与任务并发。

任务队列空

没有别的活儿在等。

空闲 ≥ 300 秒

now − last_user_activity_ms ≥ idle_threshold_sec。

当前未在打盹

再入保护。

用户活动指改变状态的 RPC（start_task、thread_new……）；轮询类如 ping、get_status 不算。打盹期间出现用户活动会立即 set() 取消标志——用户永远不会等打盹。

v0 — 文本通道

一次只读一条未处理线程，构造紧凑文本（无图），把现有 tip / 记忆摘要也喂给 LLM，仅暴露 learn_tip / remember / load_app_tips。落地到 tools.md 与 apps/<slug>/tips.md。日志按天切到 ~/.lucid/doze-YYYYMMDD.log。

v1 — 图标通道

新增工具 propose_icon(image_filename, x, y, w, h, label, description?)，处理器会：

  • 对照线程的 step_image 事件校验文件名
  • 用 PIL 裁出 PNG，与现有图标库做 dHash 对比
  • 入队 icon_proposals/<id>.png + index.json，留给 /doze 页面人工审核

图标库永远不会被打盹直接写入——只有被接受的提案才会进入。

v1 — 任务栏视觉学习

视觉第二步若"看到 diff 但分类器说不算"，会把反例写到 logs/taskbar-monitor/learn-queue.jsonl。打盹排干该队列，按 (app_slug, signature) 聚合，达阈值（默认 ≥ 3）后向 apps/<slug>/tips.md 写入 [taskbar_visual] tip。下一次第二步通过 load_app_tips 看到 tip，就不再误报同一小部件。

安全：打盹 worker 永远不能调 GUI 工具——dispatch 白名单只放它那一小撮。打盹只读 events.jsonl（已序列化，无 base64 图），不读可能含密的 messages.json。默认关闭，在 /settings → 打盹 启用。

CHAPTER 05

Skills 技能

Anthropic Agent Skills 标准格式 · 丢一个文件夹，就多一个能力

一个 skill 就是一个含 SKILL.md（YAML frontmatter + Markdown 正文）的文件夹——完全沿用 Anthropic 官方 Agent Skills 的格式。把文件夹丢到 ~/.lucid/skills/<slug>/，Lucid 自动识别。

格式

---
name: weekly-report
description: Open Outlook and draft a weekly report email.
version: "1.0"
license: MIT
source: user
---

# Weekly report

1. launch_app: outlook
2. ...

Skills vs 模板

模板 Templates技能 Skills
形态一段纯文本指令SKILL.md（YAML + 正文，可带脚本）
触发用户点"运行"用户点"使用"，或 智能体看 system prompt 自行决定
系统提示不注入每个技能注入一行：name (tag): description

渐进披露

list_skills()

列出可用技能。

read_skill(slug)

拉取完整 SKILL.md。

read_file(path)

打开 reference/*.md、scripts/*.py 等附件。

在线安装（由 LLM 引导）

README 开篇示例演示的推荐 bootstrap 路径：

run_shell("git clone https://github.com/anthropics/skills ~/.lucid/skills")
read_file("~/.lucid/skills/pptx/SKILL.md")
learn_tip("PPTX skill installed at ~/.lucid/skills/pptx; entry: build.py")
run_shell("python ~/.lucid/skills/pptx/build.py proposal.docx proposal.pptx")

RPC install_skill_url 仅允许用户主动触发（LLM 不能调），结果标记 source: "online"（不可信），由安全层决定。

CHAPTER 06

语音输入

按住说话 · 完全离线 · 焦点感知

说一句话，Lucid 自动决定是开新线程、打断当前线程、还是追加到输入框。

触发

长按 Space ≥ 5 秒

无障碍优先：默认不要组合键。

可配置

hotkey、hold_threshold_ms、stop_mode（release / tap_again / auto_silence）。

焦点感知

文本框或输入法合成中时，长按被忽略——Space 还是 Space。

ASR——本地离线

faster-whisper

其他引擎（sherpa-onnx / vosk）保留向后兼容但不进 UI。

本地模型权重

不上云，全部留本机。

≤ 1.5 秒

CPU 上 small 模型停录 → 文字。

意图分发 计划中

转写之后，一个小分类 LLM（独立短系统提示，~150 token）从三种动作里挑一：

thread_new
显然是新任务（"打开记事本写 hello"）。
thread_abort
叫停当前（"取消"、"停下"、"算了"）。
dictation_append
补充输入框（"……再加一句……"）。

置信度低时，浮层弹快选 chips，用户一步定夺。当前浮层挂在屏幕顶部居中，带波形和倒计时。

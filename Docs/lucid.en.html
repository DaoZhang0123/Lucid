CHAPTER 01

Architecture

Pure-vision agent · three-layer process model · two public LLM backends

Lucid does not depend on MCP, application plug-ins, or app APIs. The model is Claude Opus 4.x via two public backends, plus a hidden internal proxy for development.

Anthropic API · claude-opus-4-5-20250929 GitHub Copilot OAuth · claude-opus-4-6 Internal proxy (dev only)

Three-layer process model

GUI · Tauri WebView

SvelteKit pages: Chat, Schedules, Templates, Memory, Doze, Settings.

Rust shell · Tauri main

Sidecar lifecycle, system tray, global hotkey, DPAPI-encrypted credentials.

Engine · lucid.exe

Python 3.14 sidecar: ReAct loop, screenshot pyramid, taskbar, scheduler, doze, memory.

UI ↔ engine is JSON-RPC over stdio. User data lives at ~/.lucid/ (config, logs, schedules, memory, icons, token).

Why pure-vision?

Zero integration cost

Works on private ERPs, RDP sessions, old Win32 apps, even games.

HITL safety

Three autonomy tiers: full, confirm_critical, confirm_each + panic hotkey + PyAutoGUI fail-safe.

Covers everything you can see

Slower than dedicated APIs, but no integration ever goes stale.

ReAct loop

observe
Capture screen at the level the model chose (L0–L3).
think
Claude returns tool_use: computer, launch_app, region, …
safety
Safety Layer screens danger words, coordinates, HITL gate.
act
Execute via PyAutoGUI / Win32 SendInput / clipboard.
verify
Post-step L3 pixel-diff if click_verify_enabled.
repeat
Until finish() or budget hit.
CHAPTER 02

Screenshot pipeline

Four levels · token-budgeted · pixel-precise click verification

Images are expensive — don't send unless asked. The model is taught that "not seeing anything is normal; if you need to look, call screenshot()." The system attaches images only as a side-effect of a few high-value actions.

Four levels (L0–L3)

LevelScopeTriggerPurpose
L0Icon atlasscreenshot(level="icon_atlas")Identify unknown tray / taskbar icons
L1Full virtual desktopModel-initiatedCross-app situational awareness
L2Active window client areaSide-effect of launch_app / focus_window, onceForms, menus, list operations
L3~210×210 around the cursor, lime crosshairClick-verify, model-initiatedPixel-precise click confirmation, tooltip / small icon reading

Key choices

No auto post-step images

Removed in 2026-05. Saves 30–50% tokens on long threads.

L2 grid overlay

Only L2 carries a coordinate grid; system prompt teaches the model to read pixel coords from it.

L3 fixed tile + crosshair

No more UIA smart-sizing — predictable size keeps the model's coordinate mental model stable.

Click verify

L3 before/after every click — pixel_change_ratio < 0.5% warns "may have missed".

Recompression budget

Old screenshots are downscaled + JPEG'd, then replaced with a text placeholder. Forces "transcribe in the same turn".

DPI-aware coordinates

Per-Monitor V2, mss physical pixels, active-app pin defends against focus jumps.

CHAPTER 03

Taskbar notification monitor

Two parallel channels · whichever fires first wins

Action Center (WinRT UserNotificationListener) doesn't catch Teams / WeChat reliably, so Lucid runs two parallel channels.

Channel A — UIA (default, event-driven) LIVE

Property-change events

Subscribes to Name / HelpText on the Shell_TrayWnd subtree.

Zero polling, zero tokens

The event itself tells us which app and what changed.

Cooldown + snapshot sweep

Per-app cooldown so a single notification doesn't replay.

lucid/taskbar_uia.py · config [taskbar_uia]

Channel B — Visual two-step (fallback) LIVE

Diff
Every poll_interval_sec, grab the bottom-centre strip (≈ bottom 120 px, x ∈ 25–75 %). dHash / pixel-diff; below threshold → done.
LLM confirm
On a real diff, ask "Is this a new chat notification? If yes, which app?" with the cropped strip attached. Output is structured.

X-projection segmentation isolates the changed column so the time / weather widget doesn't trigger false positives.

Arbitration

  • When UIA confirms first, Step 2 LLM is suppressed for visual_suppress_after_uia_sec (default 20 s).
  • Confirmed events are dedup'd in a strict-mode queue and may trigger auto-reply via auto_chat_instruction.
  • Per-app channel preference lives in taskbar_sources.json — doze learns and writes this file (Chapter 4).
CHAPTER 04

Doze learning

Low-priority reflection on idle · learns tips, icons, taskbar widgets

v0 text — LIVE  v1 icon — LIVE  v1 taskbar-visual — LIVE  v2 heartbeat — planned

When does it wake?

Every tick_interval_sec (60 s default) the worker checks four conditions; all must hold:

No running worker

Doze never overlaps itself or task threads.

Task queue empty

Nothing else waiting.

Idle ≥ 300 s

now − last_user_activity_ms ≥ idle_threshold_sec.

Not already dozing

Re-entry guarded.

User activity means state-changing RPCs (start_task, thread_new, …). Polling like ping / get_status doesn't count. Any user activity during a pass set()s the cancel flag — the user never waits for doze.

v0 — text channel

One pass reads one unprocessed thread, builds a compact transcript (no images), shows the LLM existing tips / memory digest, and exposes only learn_tip / remember / load_app_tips. Output goes to tools.md and apps/<slug>/tips.md. Logs split per day at ~/.lucid/doze-YYYYMMDD.log.

v1 — icon channel

Adds propose_icon(image_filename, x, y, w, h, label, description?). The handler:

  • validates the filename against the thread's step_image events
  • crops the PNG with PIL, dHash-compares against the existing atlas
  • enqueues to icon_proposals/<id>.png + index.json for human review on the /doze page

The atlas is never written directly from doze — only accepted proposals make it in.

v1 — taskbar-visual learning

Visual Step 2 writes "diff seen but classifier said no" counter-examples to logs/taskbar-monitor/learn-queue.jsonl. Doze drains the queue, groups by (app_slug, signature), and after ≥ 3 examples writes a [taskbar_visual] tip into apps/<slug>/tips.md. Next Step 2 call sees the tip via load_app_tips and stops re-flagging the same widget.

Safety: the doze worker can never call GUI tools — the dispatch whitelist refuses anything outside its small tool set. Doze reads events.jsonl (no base64 images), not messages.json. Off by default; enable in /settings → Doze.

CHAPTER 05

Skills

Anthropic Agent Skills format · drop a folder, get a capability

A skill is a folder containing a SKILL.md with YAML frontmatter and a Markdown body — the exact format used by Anthropic's official Agent Skills. Drop a skill folder into ~/.lucid/skills/<slug>/ and Lucid picks it up.

Format

---
name: weekly-report
description: Open Outlook and draft a weekly report email.
version: "1.0"
license: MIT
source: user
---

# Weekly report

1. launch_app: outlook
2. ...

Skills vs Templates

TemplatesSkills
ShapePlain instruction stringSKILL.md (YAML + body, can ship scripts)
TriggerUser clicks RunUser clicks Use or agent decides from system prompt
System promptNot injectedOne line per skill: name (tag): description

Progressive disclosure

list_skills()

Enumerate available skills.

read_skill(slug)

Pull the full SKILL.md body.

read_file(path)

Open reference/*.md, scripts/*.py, any companion file.

Online install (LLM-led)

The README's opening example demonstrates the recommended bootstrap:

run_shell("git clone https://github.com/anthropics/skills ~/.lucid/skills")
read_file("~/.lucid/skills/pptx/SKILL.md")
learn_tip("PPTX skill installed at ~/.lucid/skills/pptx; entry: build.py")
run_shell("python ~/.lucid/skills/pptx/build.py proposal.docx proposal.pptx")

RPC install_skill_url is restricted to user action (not LLM-triggered) and tags the result with source: "online" (untrusted) so the safety layer can decide.

CHAPTER 06

Voice input

Push-to-talk · fully offline · focus-aware

Speak a sentence and Lucid either opens a new thread, aborts the current one, or appends to the input box.

Trigger

Hold Space ≥ 5 s

Accessibility-first: no modifier combos by default.

Configurable

hotkey, hold_threshold_ms, stop_mode (release / tap_again / auto_silence).

Focus-aware

When a text input or IME composition is active, hold is ignored — Space stays Space.

ASR — local, offline

faster-whisper

Other engines (sherpa-onnx / vosk) kept for back-compat but hidden from UI.

Local weights

No cloud round-trip; everything stays on the box.

≤ 1.5 s

Stop → text on CPU for the small models.

Intent dispatch planned

After transcription, a small classification LLM (~150 max tokens) decides one of three actions:

thread_new
Clearly a new task ("open Notepad and write hello").
thread_abort
Stop the current run ("cancel", "stop", "never mind").
dictation_append
Extend the input box ("…and also mention…").

If confidence is low, the overlay shows quick-pick chips so the user resolves it in one frame. Today the overlay sits top-centre with waveform + countdown.
